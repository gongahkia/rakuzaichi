# Rakuzaichi Product & Engineering Upgrade Notes

## 1) Repository structure, purpose, and intent

Rakuzaichi is a cross-browser WebExtension that exports active AI chat conversations into portable file formats while keeping processing local in-browser.

### Source of truth
- Core extension source lives in `src/`.
- Browser-specific bundles are generated by `scripts/build.mjs` into `dist/`.
- Safari app wrapping is handled by `scripts/convert-safari.mjs` and `safari/` project files.

### Runtime architecture
- `src/content-script.js` detects supported chatbot UIs and extracts normalized message envelopes.
- `src/platforms/*.js` contain platform-specific selectors and extractors.
- `src/background-core.js` handles download orchestration and auto-export scheduling.
- `src/popup.*` provides user-facing export actions.
- `src/options.*` controls persistent settings.
- `src/converters.js` serializes normalized chat envelopes into export formats.
- `src/storage.js` and `src/history.js` store settings/history in extension local storage.

### Engineering intent
The original repository intent is a privacy-first, no-backend tool: users keep their data and get multi-format portability without SaaS lock-in.

## 2) Philosophical value proposition assessment

### Existing value proposition (before this upgrade)
- Strong: local-first exports and broad platform coverage.
- Weak: UI hierarchy and trust signals were minimal, making value less obvious to everyday users.
- Weak: failures were surfaced as short messages but lacked traceable diagnostics context.
- Weak: settings page lacked operational visibility for debugging/maintenance.

### Upgraded value proposition (after this upgrade)
Rakuzaichi now positions itself as:

> A local-first, single-purpose AI chat export utility with transparent diagnostics, fast one-click flows for everyday users, and structured outputs/debug tooling for developers.

This strengthens both breadth (additional practical format support) and depth (operational reliability + diagnostic transparency + richer UX context).

## 3) Market and convention alignment research

This upgrade aligns with extension ecosystem conventions and successful UX patterns:

1. Single-purpose clarity is explicitly required for extension quality and user trust.
- Source: Chrome Web Store quality guidelines FAQ
- https://developer.chrome.com/docs/webstore/program-policies/quality-guidelines-faq

2. Permission minimization and privacy transparency are key adoption criteria.
- Source: Chrome extension user privacy guidance
- https://developer.chrome.com/docs/extensions/develop/security-privacy/user-privacy

3. Popups should be action-centric and optimized for quick, user-initiated flows.
- Source: MDN WebExtensions popup behavior/design notes
- https://developer.mozilla.org/en-US/docs/Mozilla/Add-ons/WebExtensions/user_interface/Popups

4. High-adoption extension UX patterns emphasize one-click core actions with clear destination/context.
- Source: Notion Web Clipper (one-click capture pattern)
- https://www.notion.com/web-clipper

5. Competing chat-export tools highlight privacy-local processing + one-click export + readable output quality.
- Source: Capsula positioning
- https://seveneves.ai/capsula/
- Source: ChatShell positioning
- https://www.chatshellexport.com/

Inference from these sources:
- The winning proposition is not "many random features". It is clear purpose, low-friction action, explicit privacy narrative, and reliable operational feedback.

## 4) What was implemented in this upgrade

### Frontend rework (heavy)
- Rebuilt popup information architecture:
  - Platform/context cards (platform, message count, title, model).
  - Quick-export CTA using default format.
  - Expanded format toolbox with use-case hints.
  - Preview panel now supports both download and copy-to-clipboard.
  - Latest export summary visible in popup.
  - Trust footer reinforces local-only handling.
- Rebuilt options page into a control center:
  - Cleaner sectioned settings UX.
  - Inline auto-export status with trace context.
  - Diagnostics panel with recent log events.
  - Download/clear diagnostics and export history controls.
- Applied a distinct visual language:
  - New gradients, panel depth, stronger hierarchy, responsive behavior.
  - Motion via staged reveal animation.

### Reliability and diagnostics depth
- Added a new shared logger module: `src/logger.js`.
  - Structured levels (`debug/info/warn/error`), trace IDs, bounded local retention.
  - Local diagnostics persistence for post-failure debugging.
- Integrated logging and trace IDs through key runtime paths:
  - `content-script` detection/extraction lifecycle.
  - `background-core` download/auto-export lifecycle.
  - popup and options initialization/actions.
- Hardened storage access failures with explicit error messages in `src/storage.js`.

### Capability breadth
- Added **Markdown** export format (`markdown` -> `.md`) in `src/converters.js`.
- Exposed Markdown in popup and options format selections.

### CI/CD hygiene
- Added `.github/workflows/ci.yml` to run:
  - install
  - build
  - unit tests
  - manifest validation
  - permission snapshot checks

## 5) User outcomes this improves

### Everyday users
- Faster understanding of whether the current tab is exportable.
- Better confidence from explicit privacy and status messaging.
- Fewer dead-end interactions from clearer unsupported-tab guidance.

### Developers and power users
- Markdown export for docs and engineering workflows.
- Structured local diagnostics with trace IDs for reproducible bug reports.
- Easier auditing via downloadable diagnostics/history JSON.
